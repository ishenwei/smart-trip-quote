# Smart Trip Quote 智能旅行报价系统用户使用说明书

## 前言
本说明书旨在为无技术背景的用户提供Smart Trip Quote智能旅行报价系统的完整操作指南。通过图文结合的方式，详细介绍从旅游需求创建到行程导出的全部流程，帮助用户轻松完成个性化旅游行程规划和报价获取。

## 一、系统概述
Smart Trip Quote（智能旅行报价）系统是一款集旅游需求管理、智能行程规划、行程内容管理和文档导出功能于一体的应用平台。系统利用先进的大语言模型（LLM）技术，支持用户通过自然语言描述旅行需求，自动提取结构化信息并生成精准的旅行报价。用户可通过直观的界面操作，完成从需求提出到行程落地的全过程。

### 系统核心特点
- **智能化处理**：利用LLM技术实现自然语言理解，简化用户输入流程
- **多提供商集成**：支持多个LLM提供商（DeepSeek、Gemini、OpenAI），提高系统可靠性
- **个性化推荐**：根据用户需求生成定制化的行程方案
- **实时报价**：快速计算并提供详细的旅行费用明细
- **多渠道接入**：支持Web界面和API接口，满足不同场景需求

## 二、用户端功能操作指南

### 2.1 旅游需求创建
1. **登录系统**：在系统登录页面输入用户名和密码，点击"登录"按钮进入系统首页
2. **进入需求创建页面**：在首页导航栏点击"创建旅游需求"选项，进入需求创建界面
3. **填写旅游需求信息**：
   - **自然语言输入**：在文本框中用自然语言描述您的旅行需求，例如："我想从北京去上海旅游5天，2个成人，预算5000到10000元，想住舒适型酒店，坐高铁去"
   - **表单输入**：也可以通过传统表单方式填写详细信息：
     - 出发地选择：从下拉菜单中选择出发城市
     - 目的地选择：从下拉菜单中选择旅游目的地城市
     - 出行日期：通过日期选择器设置旅游开始和结束日期
     - 同行人数：输入或通过加减按钮选择同行人数（成人、儿童、老人）
     - 预算范围：拖动滑块或直接输入预算金额范围
     - 交通方式：选择大交通方式（如双飞、高铁、自驾等）
     - 酒店等级：选择酒店等级（经济型、舒适型、高档型、豪华型）
     - 旅游偏好：勾选感兴趣的旅游类型（如自然风光、历史文化、美食体验等）
     - 特殊需求：在文本框中输入其他特殊要求（如无障碍设施需求、饮食禁忌等）
4. **保存需求**：填写完成后，点击页面底部"保存需求"按钮，系统将自动保存您的旅游需求

### 2.2 需求转换为旅游行程规划
1. **查看已保存需求**：在"我的需求"页面中，找到需要转换的旅游需求
2. **生成行程规划**：点击需求卡片右下角的"生成行程"按钮，系统将自动进入行程生成流程
3. **等待智能分析**：系统将利用大语言模型分析您的需求，匹配最佳行程方案，此过程可能需要1-3分钟
4. **查看初步行程**：生成完成后，系统将自动跳转至行程预览页面，展示初步行程规划和报价明细

### 2.3 行程规划查看与调整
1. **整体行程概览**：在行程预览页面，可查看整个行程的日期安排、行程亮点和总预算
2. **每日行程详情**：点击"查看每日行程"按钮，展开查看每一天的详细安排
3. **行程调整功能**：
   - 调整顺序：通过拖拽方式调整每日行程顺序
   - 修改内容：点击具体行程项目（如景点、酒店），在弹出的修改窗口中更换选项
   - 添加/删除项目：点击"添加项目"按钮增加新的行程内容，或点击项目旁的删除图标移除项目
   - 调整预算：根据需要调整行程预算，系统将自动重新计算报价
4. **保存行程修改**：完成调整后，点击"保存修改"按钮保存当前行程状态

### 2.4 行程报价查看
1. **报价明细**：在行程详情页面，查看详细的报价明细，包括交通、住宿、餐饮、景点门票等费用
2. **价格对比**：系统会提供不同方案的价格对比，帮助您做出最优选择
3. **价格调整**：如果对报价不满意，可以返回修改行程内容，系统会实时更新报价

## 三、管理员界面操作指南

### 3.1 管理员登录与权限
1. 使用管理员账号登录系统，进入管理员专属界面
2. 管理员界面包含行程管理、数据管理、系统设置和LLM服务管理四个主要功能模块

### 3.2 旅游行程规划管理
1. **行程列表查看**：在"行程管理"页面，可查看系统中所有用户创建的行程规划
2. **行程审核**：对用户提交的行程规划进行审核，点击"审核"按钮可查看详情并批准或退回
3. **行程推荐**：对优质行程进行标记，使其在用户推荐列表中展示
4. **行程数据分析**：查看行程创建数量、热门目的地等统计数据

### 3.3 每日行程管理
1. 在"数据管理"菜单下选择"每日行程管理"
2. 查看系统中所有的每日行程模板
3. 点击"添加模板"创建新的每日行程模板
4. 对现有模板进行编辑、复制或删除操作

### 3.4 景点数据管理
1. 在"数据管理"菜单下选择"景点管理"
2. 查看系统中的所有景点信息，包括名称、位置、简介、门票价格等
3. 点击"添加景点"按钮录入新的景点信息
4. 对现有景点信息进行编辑、更新或删除操作
5. 批量导入/导出景点数据

### 3.5 酒店数据管理
1. 在"数据管理"菜单下选择"酒店管理"
2. 查看系统中的所有酒店信息，包括名称、位置、价格、星级、设施等
3. 添加新酒店信息或对现有酒店信息进行维护
4. 设置酒店与景点的关联关系

### 3.6 餐厅数据管理
1. 在"数据管理"菜单下选择"餐厅管理"
2. 查看系统中的所有餐厅信息，包括名称、菜系、价格区间、用户评分等
3. 添加新餐厅信息或对现有餐厅信息进行更新维护
4. 根据用户反馈调整餐厅推荐优先级

### 3.7 定价规则管理
1. 在"数据管理"菜单下选择"定价规则管理"
2. 查看和编辑现有的定价规则
3. 添加新的定价规则，设置不同季节、不同目的地的价格调整策略
4. 预览定价规则的应用效果

### 3.8 LLM服务管理
1. 在"系统设置"菜单下选择"LLM服务管理"
2. 配置LLM提供商的API密钥和参数
3. 查看LLM服务的使用统计和性能指标
4. 调整LLM服务的缓存和速率限制设置
5. 测试不同LLM提供商的响应效果

## 四、行程导出功能

### 4.1 导出准备
1. 在行程详情页面，确认所有行程内容已完善
2. 检查行程中的日期、景点、酒店、餐厅等信息是否准确

### 4.2 选择导出格式
1. 点击行程页面右上角的"导出行程"按钮
2. 在弹出的导出选项中，选择所需的文档格式（如PDF、Word、Excel等）
3. 选择导出内容范围（完整行程、仅每日安排、仅住宿信息、仅报价明细等）

### 4.3 执行导出
1. 点击"确认导出"按钮，系统开始生成文档
2. 等待文档生成完成（通常需要10-30秒）
3. 生成完成后，系统将自动下载文档到本地或提供在线查看选项

### 4.4 导出文档使用
1. 在本地找到下载的行程文档并打开
2. 可打印文档随身携带，或保存到移动设备中随时查看
3. 文档包含行程的所有详细信息及必要的联系方式

## 五、API接口使用指南

### 5.1 接口概述
Smart Trip Quote系统提供了完整的RESTful API接口，支持第三方系统集成和自动化操作。

### 5.2 主要接口

#### 5.2.1 处理旅游需求接口
- **接口路径**: `/api/process/`
- **请求方法**: `POST`
- **功能描述**: 处理用户提交的旅行需求，使用LLM技术分析自然语言输入并提取结构化数据

**请求示例**:
```json
{
  "user_input": "我想在明年3月份去日本东京旅行，为期5天，预算在5000元左右",
  "provider": "openai",
  "client_id": "client_123",
  "save_to_db": true
}
```

#### 5.2.2 其他接口
- **提供商信息**: `/api/provider-info/` (GET)
- **速率限制统计**: `/api/rate-limit-stats/` (GET)
- **缓存统计**: `/api/cache-stats/` (GET)
- **清除缓存**: `/api/cache/clear/` (POST)
- **重载配置**: `/api/config/reload/` (POST)
- **健康检查**: `/api/health/` (GET)

### 5.3 API文档
启动服务后，可访问以下地址查看完整的Swagger API文档：
```
http://localhost:8000/swagger/
```

## 六、常见问题解答

### 6.1 基本操作问题

**Q1: 如何使用自然语言输入创建旅游需求？**
A: 在需求创建页面的文本框中，用自然语言描述您的旅行需求，例如："我想从北京去上海旅游5天，2个成人，预算5000到10000元，想住舒适型酒店，坐高铁去"。系统会自动分析并提取结构化信息。

**Q2: 生成行程需要多长时间？**
A: 通常需要1-3分钟，具体时间取决于网络状况和LLM服务的响应速度。

**Q3: 如何调整行程预算？**
A: 在行程详情页面，您可以修改行程内容（如酒店等级、交通方式等），系统会实时更新报价。也可以直接调整预算范围，系统会根据新的预算重新规划行程。

### 6.2 技术问题

**Q1: 系统提示"LLM服务不可用"怎么办？**
A: 这可能是由于网络连接问题或LLM提供商服务暂时中断。请稍后重试，或联系系统管理员。

**Q2: 导出文档失败怎么办？**
A: 请检查网络连接，确保浏览器允许弹出窗口和文件下载。如果问题持续，请联系客服支持。

**Q3: 如何提高需求分析的准确性？**
A: 提供更详细的自然语言描述，包括出发地、目的地、出行日期、人数、预算、偏好等关键信息。系统会根据这些信息提供更准确的分析结果。
