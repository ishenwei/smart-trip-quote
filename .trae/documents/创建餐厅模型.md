# 创建餐厅模型

## 实现步骤

1. **创建餐厅模型文件**
   - 在 `apps/models/` 目录下创建 `restaurant.py` 文件
   - 定义 `Restaurant` 类，继承自 `BaseModel`

2. **定义模型字段**
   - 根据提供的表结构定义所有字段
   - 使用 `UUIDField` 作为 `restaurant_id` 主键
   - 使用 `CharField` 定义字符串字段
   - 使用 `TextField` 定义文本字段
   - 使用 `BooleanField` 定义布尔字段
   - 使用 `IntegerField` 定义整数字段
   - 使用 `DecimalField` 定义小数字段
   - 使用自定义的 `JSONField` 存储 JSON 数据
   - 使用 `TextChoices` 定义枚举类型字段

3. **定义枚举类型**
   - 定义 `RestaurantType` 枚举：FINE_DINING, CASUAL, FAST_FOOD, CAFE, BAR, STREET_FOOD, BUFFET, OTHER
   - 定义 `PriceRange` 枚举：$, $$, $$$, $$$$
   - 定义 `Status` 枚举：ACTIVE, INACTIVE, TEMPORARILY_CLOSED, PERMANENTLY_CLOSED

4. **添加验证器**
   - 为评分字段添加 `MinValueValidator` 和 `MaxValueValidator`，范围 0-5

5. **定义 Meta 类**
   - 设置 `db_table` 为 'restaurants'
   - 设置 `verbose_name` 和 `verbose_name_plural`
   - 添加适当的索引

6. **添加方法**
   - 实现 `__str__` 方法
   - 实现 `clean` 方法进行数据验证

7. **更新 __init__.py**
   - 在 `apps/models/__init__.py` 中导入 `Restaurant` 模型

## 技术要点

- 使用 `UUIDField` 作为主键，需要导入 `uuid` 模块
- 使用自定义的 `JSONField` 存储 JSON 数据
- 使用 `TextChoices` 定义枚举类型，提高代码可读性
- 为评分字段添加范围验证
- 遵循项目现有的代码风格和模型设计模式
- 确保字段类型和约束与提供的表结构一致
